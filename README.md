## 访问方式与工作原理

本项目采用前后端分离与本地优先的设计理念，提供两种的访问方式：

### 1. 访问方式

本应用的核心 UI 为纯静态单页应用，支持以下两种方式无缝使用：

- **Android 客户端**：通过原生 WebView 进行封装，提供 App 体验。
- **Web 网页端**：无需下载安装，直接通过浏览器访问网址 [schedule.jiraki.top](https://schedule.jiraki.top) 即可使用。

### 2. 本地优先

无论是使用 App 还是网页端，应用都极度依赖本地存储。

- **资源与数据全缓存**：核心代码和 UI 静态资源在首次加载后即被缓存。更重要的是，您的课表、成绩等教务数据在获取后，会被永久持久化在设备本地。
- **纯离线体验**：日常的课表查看、单双周切换、成绩查阅、自定义课程管理等功能，均为**完全离线操作**。这些操作不消耗任何网络流量，响应速度达到毫秒级零延迟。

### 3. 按需联网与云端解析

应用仅在用户主动触发以下几种特定需求时，才会发起网络请求并连接教务处：

- **数据同步**（拉取最新课表、成绩、考试信息）
- **空闲教室**
- **蹭课查询**
- **图书查询**

### 4.后端服务说明

为了维持项目的免费运作，网页端解析服务部署于 Render 免费实例，受限于平台规则，存在以下客观情况：

* **冷启动延迟**：若服务器连续十几分钟无访问，会自动进入休眠状态。因此，**首次点击“同步数据”时，可能需要等待 40 至 60 秒的服务器唤醒时间**。一旦唤醒，后续的查询请求将恢复秒级响应，请耐心等待。
* **并发限制**：免费服务器内存极小，在期末查分等高并发时期极易过载。若请求失败，请稍候再试，避免频繁点击重试对服务器造成二次冲击。

## 本地运行与二次开发

本项目逻辑极度解耦，非常适合前端开发者进行二次开发与定制。

### 1. 网页端开发
前端代码位于 `front/` 目录下。无需配置复杂的 Node.js 编译环境，直接使用浏览器打开 `index.html` 即可预览 UI 与交互逻辑。
*注：本地调试教务接口时，需自行配置代理以解决跨域问题。*

### 2. 安卓端打包
本项目安卓打包部分未上传本仓库。仅需简单修改几行配置打包即可。

## 开发初衷与致敬

对于许多南理工学子而言，「周三课表」小程序曾是大学生活中不可或缺的教务工具。然而，随着开发者毕业步入社会，以及学校教务系统风控策略的不断收紧，个人维护的成本难以为继。4月7日，该小程序正式发布了停运通知。在此，向这位前人栽树的开拓者致以敬意。

巧合的是，在停运前几日（约4月3日），因察觉到现有工具的服务波动，我萌生了制作一个备用课表的念头。借助 AI 辅助编程，我用数天时间构建了本工具的雏形。原本这只是一项自用方案，恰逢「周三课表」谢幕，我决定将项目进一步完善并开源发布。

薪火相传，希望这个纯粹、无广告的工具能填补空缺，继续为南理工的同学们提供便利。
